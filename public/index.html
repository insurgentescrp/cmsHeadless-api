Headless CMS API

API REST para gestión dinámica de contenido

http://localhost:3000
Auth Content Types Fields Entries Authors Media Filters

🔐 Auth NUEVO

Registro, inicio de sesión y perfil de usuario

POST /api/auth/register
Crear una cuenta nueva
{
  "username": "juanperez",        // requerido
  "email": "juan@example.com",    // requerido
  "password": "mi_password",      // requerido
  "firstName": "Juan",
  "lastName": "Pérez"
}
POST /api/auth/login
Iniciar sesión y obtener JWT
{
  "email": "juan@example.com",
  "password": "mi_password"
}
Respuesta:
{
  "success": true,
  "data": {
    "user": { "id": "uuid", "username": "juanperez", ... },
    "token": "eyJhbGciOiJIUzI1Ni..."
  }
}
GET /api/auth/me 🔒 requiere token
Obtener perfil del usuario autenticado
curl -H "Authorization: Bearer <token>" http://localhost:3000/api/auth/me

🔑 Usar el token: incluir en el header Authorization: Bearer <token> en las rutas protegidas.

📦 1. Content Types

Definen los modelos de contenido (posts, pages, productos, etc.)

POST /api/content-types
Crear un content type
{
  "name": "post",             // requerido, único
  "displayName": "Post",      // si se omite, usa name
  "description": "Blog posts",
  "isSingle": false
}
GET /api/content-types
Listar todos los content types
GET /api/content-types/:id
Obtener un content type por ID
DELETE /api/content-types/:id
Eliminar (cascade: elimina fields y entries asociados)

⚙️ 2. Fields

Definen la estructura/atributos de cada content type

POST /api/fields
Crear un campo
{
  "contentTypeId": "uuid",         // requerido
  "name": "title",                  // requerido
  "displayName": "Título",
  "fieldType": "string",            // requerido
  "isRequired": true,
  "isList": false,
  "defaultValue": null,
  "options": null,                  // {"values":["a","b"]} para selects
  "position": 0
}
GET /api/fields/content-type/:contentTypeId
Obtener campos de un content type
GET /api/fields/:id
Obtener un campo por ID
PATCH /api/fields/:id
Actualizar parcialmente un campo
DELETE /api/fields/:id
Eliminar un campo

🧠 Tipos de campo soportados

string text richtext number boolean date image relation

📄 3. Content (Entries)

Contenido dinámico (usar el nombre del content type como endpoint)

Las entradas se validan automáticamente contra los fields definidos en el content type.

POST /api/:contentType
Crear entrada (ej: POST /api/post)
curl -X POST http://localhost:3000/api/post \
  -H "Content-Type: application/json" \
  -d '{"title":"Hola mundo", "content":"Mi primer post"}'
GET /api/:contentType
Listar entradas del content type
GET /api/:contentType/:id
Obtener entrada por ID
PATCH /api/:contentType/:id
Actualizar entrada (merge parcial en JSONB)
DELETE /api/:contentType/:id
Eliminar entrada (soft delete)

✍️ Authors NUEVO

Gestión de autores/periodistas con asignación a entries

POST /api/authors
Crear un autor
{
  "name": "Carlos Pérez",
  "slug": "carlos-perez",
  "bio": "Editor general del periódico",
  "avatarUrl": "https://imgur.com/avatar.jpg",
  "email": "carlos@example.com",
  "role": "editor",              // reporter | columnist | photographer | editor
  "twitter": "@carlosperez",
  "instagram": "@carlosperez"
}
GET /api/authors
Listar todos los autores
GET /api/authors/:id
Obtener autor por ID (incluye entries asociadas)
PATCH /api/authors/:id
Actualizar autor
DELETE /api/authors/:id
Eliminar autor

🔗 Asignar autor a una entry

POST /api/entry-authors
Asigna un autor a una entrada con posición
{
  "entryId": "uuid",
  "authorId": "uuid",
  "position": 0    // orden de firma
}

📋 Consultar entries con autores

SELECT
  e.slug,
  e.data->>'title' AS title,
  jsonb_agg(jsonb_build_object('name', a.name, 'role', a.role)) AS autores
FROM entries e
JOIN entry_authors ea ON ea.entry_id = e.id
JOIN authors a ON a.id = ea.author_id
GROUP BY e.id;

🎬 Media NUEVO

Gestión de medios externos (YouTube, imágenes en hosting externo, etc.)

POST /api/media
Registrar un medio externo
{
  "externalUrl": "https://www.youtube.com/watch?v=abc123",
  "provider": "youtube",       // youtube | vimeo | imgur | custom
  "mediaType": "video",         // video | image | audio | embed
  "title": "Video reportaje",
  "caption": "Entrevista exclusiva",
  "alt": "descripción para accesibilidad",
  "credits": "Foto: Carlos Pérez",
  "width": 1920,
  "height": 1080
}
GET /api/media
Listar todos los medios
GET /api/media/:id
Obtener medio por ID
PATCH /api/media/:id
Actualizar medio
DELETE /api/media/:id
Eliminar medio

🔗 Asociar medio a una entry

POST /api/entry-media
Vincula un medio a una entrada
{
  "entryId": "uuid",
  "mediaId": "uuid",
  "usage": "featured",    // featured | gallery | embed
  "position": 0
}

🔎 Filtros dinámicos

Las entries se pueden filtrar por cualquier campo del JSONB via query params

GET /api/post?title=hola
Filtrar entradas donde title contenga "hola" (ILIKE)
GET /api/post?status=published
Filtrar por estado
GET /api/post?status=published&author=juan
Múltiples filtros simultáneos

🚀 Ejemplo completo con curl

1. Registrarse 
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","email":"admin@ejemplo.com","password":"123456"}'
2. Iniciar sesión (guardar token) 
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@ejemplo.com","password":"123456"}'
3. Crear content type 
curl -X POST http://localhost:3000/api/content-types \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer TOKEN" \
  -d '{"name":"post","displayName":"Post","description":"Artículos"}'
4. Crear fields 
curl -X POST http://localhost:3000/api/fields \
  -H "Content-Type: application/json" \
  -d '{"contentTypeId":"UUID","name":"title","displayName":"Título","fieldType":"string","isRequired":true}'
5. Crear entrada 
curl -X POST http://localhost:3000/api/post \
  -H "Content-Type: application/json" \
  -d '{"title":"Mi artículo","content":"Texto del artículo"}'
6. Consultar entradas 
curl http://localhost:3000/api/post
